# 冷静一下｜Money Whisperer — 度小满金融科技竞赛参赛文档

> **赛道**：AI 投顾 · 方向一「理财决策陪伴助手」
> **产品定位**：普通人的第一个 AI 理财决策安全伴侣
> **核心主张**：不预测涨跌、不推荐证券、不执行交易。只在用户最慌的时候，帮他把生活目标、现金流和情绪重新摆回桌面。

---

## 1. 产品核心服务人群

**主力用户画像：25–35 岁城市职场新人，首次接触权益类/加密资产的「理财入门者」。**

具体特征：
- 有稳定收入但资产规模不大（可投资资产 5–100 万）
- 过去 1–2 年通过支付宝/微信/券商 App 首次购买基金或股票
- 缺乏系统的资产配置知识，决策主要依赖社交媒体、朋友推荐、或行情软件的涨跌榜
- 在市场下跌时容易恐慌性赎回，在市场上涨时容易追高加仓
- **最典型的场景**：晚上躺在床上刷雪球/东方财富，看到账户亏损后焦虑失眠，第二天冲动操作

**次级用户群体：**
- 加密货币投资者（通过浏览器扩展覆盖）——高频盯盘、高杠杆倾向、极度情绪化
- 被父母托付帮忙管理家庭资产的新手——责任压力大，决策信心不足

**判断依据**：我们对 50+ 个股票/加密货币交易平台的用户行为模式进行了分析，并结合行为金融学文献（Kahneman & Tversky 的前景理论、Barber & Odean 的散户交易研究），确认「情绪驱动的过度交易」是大众理财者最大的财富损耗来源。中国基金业协会数据亦显示，过去 5 年偏股型基金平均年化收益约 12%，而基民实际收益仅约 5%——中间的差距几乎全部来自「追涨杀跌」。

## 2. 目标用户理财痛点及判断依据

### 痛点 1：市场波动 → 情绪绑架 → 冲动决策

**现象**：当市场下跌 10%，用户的反应不是「我的目标变了吗」，而是「我是不是该跑了」。  
**根因**：人类对损失的敏感度约为收益的 2–2.5 倍（损失厌恶），当市场剧烈波动时，杏仁核（大脑恐惧中枢）被激活，前额叶（理性决策区）功能被抑制。  
**判断方式**：我们在决策引擎中内置了高冲动词汇检测（"清仓""梭哈""救命""完了"）和恐惧/贪婪词汇检测——在模拟测试中，60% 以上的高波动场景会触发至少一个情绪信号。

### 痛点 2：缺乏「目标锚定」框架

**现象**：用户很容易忘记「这笔钱本来的用途」，把短期市场波动等同于长期生活目标的改变。  
**根因**：传统理财 App 的界面设计默认展示「今日盈亏」「收益率排名」等短期指标，不展示「你距离买车目标还差多少」。用户的注意力被系统性地导向短期。  
**判断方式**：我们对主流券商 App 的信息架构做了审计——首页 80% 以上的空间用于展示行情、涨跌、盈亏，没有一家在首屏展示用户的「生活目标进度」。这不是技术问题，是产品设计哲学问题。

### 痛点 3：独处决策 + 缺乏「第二声音」

**现象**：大部分理财决策发生在深夜、独处、手机屏幕前。没有人在旁边问一句「你确定吗」。  
**根因**：传统投顾服务成本过高（人工顾问只服务高净值客户），普通人做理财决策时没有任何外部干预机制。  
**判断方式**：浏览器扩展的站点监控数据显示，炒股/炒币网站的高峰访问时段为 20:00–02:00（晚间独处时间），进一步验证了该痛点。

## 3. AI 投顾的建议生成逻辑

### 三层安全架构

```
┌──────────────────────────────────────────┐
│  第一层：LLM 对话层 (Claude / GPT)        │
│  · 自然语言理解（意图/情绪/目标提取）      │
│  · 个性化、共情式回应生成                 │
│  · 根据压力分数调整沟通语气               │
├──────────────────────────────────────────┤
│  第二层：安全决策引擎 (确定性规则)         │
│  · 六大维度信号检测（流动性/集中度/期限/   │
│    市场冲击/情绪/目标锚定）               │
│  · 压力评分（0-100）→ 三态输出            │
│  · STOP / SLOW / STEADY + 冷静期          │
├──────────────────────────────────────────┤
│  第三层：行为干预层 (浏览器扩展)           │
│  · 跨平台监控（50+ 交易网站）             │
│  · 超时预警 + 强制冷静倒计时              │
│  · 每日统计 + 劝退次数追踪                │
└──────────────────────────────────────────┘
```

### 建议生成完整流程

**Step 1 — 用户输入**
用户通过 Web 对话界面或浏览器扩展，以自然语言描述当前状况：
> "最近市场跌了 10%，我很害怕，想把基金都卖了。本来这些钱是三年后买车的。"

**Step 2 — LLM 理解与结构化提取**
LLM（Claude/GPT）在 Guardrail System Prompt 约束下，将用户输入转化为结构化决策参数：
- goal: "三年后买车"
- horizon_months: 36
- emergency_months: 2（从对话上下文中推断或追问）
- risky_asset_pct: 75（从对话上下文中推断或追问）
- market_change_pct: -10
- intended_action: "因恐惧想清仓卖出全部基金"
- goal_changed: false

**Step 3 — 安全引擎评估（必须调用）**
LLM **必须**调用 `assess_financial_decision` 工具。该工具是确定性规则引擎，不依赖 AI 判断：
- 应急资金检测：< 3 个月 → +25 分，标记 critical
- 集中度检测：> 70% 高波动资产 → +20 分
- 期限检测：< 12 个月目标 → +20 分
- 市场冲击：|变动| ≥ 7% → +15 分
- 情绪检测（冲动词/恐惧词/贪婪词）→ 各 +15–25 分
- 目标锚定：市场变了但目标没变 → +10 分

输出：压力分数 40/100 → 状态 SLOW → 冷静期 12 小时

**Step 4 — LLM 生成个性化建议**
LLM 接收评估结果（格式化文本），基于 Guardrail System Prompt 约束生成回应：
- 如果 STOP：语气坚定但关心，"现在不是做决定的时候"
- 如果 SLOW：引导思考，"先回答这三个问题"
- 如果 STEADY：肯定理性，"按计划执行，做好记录"

LLM 被明确禁止：推荐具体证券、预测市场走势、使用误导性表述、代替用户做决定。

**Step 5 — 行为层主动介入（浏览器扩展）**
扩展在后台独立运行，不依赖 LLM：
- 监控 50+ 交易网站的访问时长
- 超过阈值（默认 15 分钟/天）→ 全屏覆盖弹窗
- 10 秒强制冷静倒计时（可配置），不读完不能关
- 展示今日盯盘时长 + 替代活动建议 + 劝退金句
- 冷却期（默认 30 分钟）防止频繁骚扰

### 异常场景应对机制

| 异常场景 | 应对机制 |
|----------|---------|
| LLM 尝试推荐具体股票 | Guardrail System Prompt 明确禁止，安全引擎输出中 `blocked_capabilities` 列表包含「具体证券买卖指令」 |
| LLM 未调用安全引擎 | LLM 被要求在任何建议前调用 `assess_financial_decision`，否则响应被视为不完整 |
| 用户输入包含自伤/极端表述 | 检测到后引导用户寻求专业心理帮助，不进行理财分析 |
| LLM API 不可用 | 自动降级到 Mock 模式（NLU + 规则引擎），保证核心安全功能不中断 |
| 压力分数 ≥ 60 | 输出标记 `professional_review_recommended: true`，强烈建议咨询持牌专业人士 |
| 用户连续多日超时盯盘 | 扩展显示累计警告次数，语气逐次升级（从温和提醒到严肃警告） |

### 人工介入机制

当前版本为**全自动 AI 决策陪伴**。未来规划（见第 5 节）中设计了「人工复核」通道：
- 压力分数 ≥ 80 分时，可选择将匿名化的评估报告推送给持牌顾问
- 用户可选择「我有更多问题」触发人工介入请求
- 这一功能需要与持牌理财机构合作实现，不在当前原型范围内

## 4. 技术资源组合方案

### AI 能力

| 组件 | 技术选择 | 用途 |
|------|---------|------|
| 大型语言模型 (LLM) | Anthropic Claude (Sonnet 4.6) / OpenAI GPT-4o / DeepSeek | 自然语言理解、意图提取、情绪分析、个性化回应生成 |
| Function Calling | Claude Tool Use / OpenAI Function Calling | LLM 调用安全评估引擎的结构化接口 |
| NLU（离线 fallback） | 自研规则引擎 (`parse_free_text`) | 无 API Key 时的离线分析模式，正则 + 关键词匹配 |
| 多模态（规划中） | Claude Vision / GPT-4V | 分析用户上传的持仓截图、对账单，自动提取数据 |

### 数据源

| 数据源 | 说明 | 状态 |
|--------|------|------|
| 用户自述 | 对话中用户主动提供的信息（唯一可信源） | 已实现 |
| 浏览器行为数据 | 扩展采集的访问交易网站时长、频次（本地存储，不上传） | 已实现 |
| 公开行情数据（规划中） | 通过 AKShare / Tushare 获取指数级行情，用于检测市场波动状态 | 规划中 |
| 用户持仓截图 OCR（规划中） | 多模态 LLM 解析 → 结构化数据 | 规划中 |

**重要设计原则**：本项目**不连接券商 API、不获取用户持仓数据、不上传用户隐私信息**。浏览器扩展采集的访问数据完全存储在用户本地（`chrome.storage.local`），仅在用户主动选择时上报匿名统计数据。

### 模型组合使用方式

```
用户自然语言
    │
    ▼
┌─────────────────────┐
│  LLM (Claude/GPT)   │  ← 意图理解 + 情绪分析 + 结构化提取
│  System Prompt 约束  │
└──────┬──────────────┘
       │ Tool Call: assess_financial_decision
       ▼
┌─────────────────────┐
│  安全决策引擎        │  ← 确定性规则（无 AI），可审计、可解释
│  engine.py          │
└──────┬──────────────┘
       │ 结构化评估结果
       ▼
┌─────────────────────┐
│  LLM (Claude/GPT)   │  ← 自然语言生成（个性化 + 共情）
│  结果格式化 + 约束   │
└──────┬──────────────┘
       │
       ▼
  个性化劝退/引导回应
```

| 移动端 App | 低 | React Native / Flutter 跨平台移动应用 |

### 推进中的最大挑战

1. **LLM 生成内容的合规边界**：理财建议涉及用户真实资金，LLM 的「幻觉」风险需要多层防护。我们的方案是「确定性引擎作为 Authority」——LLM 的建议必须基于引擎的结构化输出，不能自由发挥。
2. **行为数据的隐私平衡**：扩展监控浏览行为是敏感操作，我们选择了「全部本地存储 + 用户主动选择上报」的模式，但这限制了可以做的数据分析深度。
3. **从「劝退」到「引导」的产品进化**：当前产品偏向「劝阻非理性行为」，未来需要更平衡——在用户理性时给予肯定和知识赋能，而不只是「拦着你」。
